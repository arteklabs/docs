Task Runner#

About#

The section shows the invoke python task runner. You can list the available tasks by running:

inv --list

Example

$ inv --list
Available tasks:

docs      Documentation Operations
lint      Lint the |project|'s source code.
quality   Run quality metrics.
test      Run unit tests.

Requirements#

  • installation

Getting Started#

The tasks are defined at tasks.py and are listed below:

tasks.docs(step='build', port=docker_host_sphinx_server_default_port, verbose=False)[source]#

Documentation Operations

Documentation#

The task docs encapsulates all the logic related to developing, building, testing and publishing the project docs.

Build#

Builds the docs and checks for WARNINGS or ERRORS.

inv docs --step build [--verbose]

Example

$ inv docs --step build
2023-02-20 18:25:16,293 INFO: (tasks.py:256)

    build ✔️

Preview#

Builds the docker image and runs the container to preview the docs with a contaneirized sphinx server listening on 0.0.0.0:port. Preview the docs with:

inv docs --step preview --port {locally-available-port}

Example

$ inv docs --step build
inv docs --step preview
local sphinx run: OK
container: dmacli_docs
host: http://0.0.0.0:8000
stop: inv docs --stop

Clean#

Remove all the static pages.

Stop Preview#

Stop the local preview container with a sphinx python server.

inv docs --step stop-preview

Example

$ inv docs --step stop-preview
dmaclidocs
param step:

The docs tasks step: build, publish, preview, stop-preview,

deploy`, destroy. Defaults to build. :type step: str, optional :param verbose: Log verbose information. Defaults to False. :type verbose: bool, optional
